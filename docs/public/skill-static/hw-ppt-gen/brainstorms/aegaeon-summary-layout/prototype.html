总结

场景：94.1% 长尾模型仅贡献 1.35% 请求，却占用 17.7% GPU。

机制：token 粒度抢占换模，绕开 request-level HOL blocking。

结果：1,192 个 H20 降至 213 个，70 小时未观察到 SLO violation。

场景：市场型 GPU 浪费
场景：长尾与突发并存
  • 长尾错配：低频模型请求少，却长期占用固定 GPU 预算。
  • 突发冗余：热门 270B 模型 burst 会越过 reserved capacity。
  • 判断：GPU pooling 的起点是市场形态，不是单模型吞吐。

平台含义：长尾和突发看似相反，本质都是用预留资源购买 SLO 安全垫。
