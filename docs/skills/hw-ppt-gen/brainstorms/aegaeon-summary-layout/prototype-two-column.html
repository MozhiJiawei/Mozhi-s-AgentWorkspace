总结

问题：长尾低频占卡与热门 burst 同时推高 GPU 预留。

判断：先证明市场形态，再证明 token-level pooling 是否值得评估。

边界：不外推到所有 LLM serving，限定为模型市场型并发服务。

左栏：市场浪费证据
长尾模型与热门 burst 共同制造预留浪费
  • 长尾错配：94.1% 模型仅贡献 1.35% 请求，却占用 17.7% GPU。
  • 突发冗余：热门 270B 模型峰值超过 reserved capacity。
  • 设计含义：大图优先保证可读，解释文字压成 3 条判断。

二分栏规则：当主证据图较宽且需要解释时，左栏给图更多高度，文字只承担读图指导。
