# 能力展示

`send-qq-email` 把邮件发送需求转成可检查的 `.eml` 快照、dry-run 结果或真实发送记录，让 Agent 发邮件这件事变得可预览、可确认、可追踪。

| 输入 | 输出 | 最关键效果 |
| --- | --- | --- |
| 收件人、主题、正文、是否真实发送 | `.eml` 快照、结构化发送结果、失败原因 | 发送前后都有可检查记录 |

![QQ 邮件发送展示](./assets/send-email-showcase.png)

这张图展示的是一次真实发送记录。对用户来说，重点不是 SMTP 细节，而是 Agent 不会悄悄发出一封不可追踪的邮件。

## 适合场景

- 发送测试邮件，确认 QQ 邮箱 SMTP 配置可用。
- 发送日报、通知或整理好的材料。
- 发送 PPT、论文分析结果或其他附件。
- 发送前先 dry-run，检查标题、正文和附件。

## 使用方式

你可以直接说：

```text
请把这段内容用 QQ 邮箱发给指定收件人，发送前先给我确认。
```

或者：

```text
请用 QQ 邮箱预演一封测试邮件，不要真实发送。
```

## 处理过程

1. 读取收件人、主题、正文和发送模式。
2. 检查 SMTP 配置是否齐全。
3. 生成 `.eml` 快照，方便发送前确认。
4. 根据用户要求 dry-run 或真实发送。
5. 返回结构化结果，说明是否发送、快照位置和失败原因。

## 交付物

| 交付物 | 用途 |
| --- | --- |
| `.eml` 快照 | 发送前检查邮件内容 |
| 发送结果 JSON | 判断是否真实发送、是否失败 |
| 错误原因 | 排查 SMTP、收件人或配置问题 |

## 展示覆盖

| Case | 输入 | 输出 | 证明的能力 |
| --- | --- | --- | --- |
| QQ 邮件发送记录 | 邮件正文、收件人、发送配置 | `.eml` 快照和发送结果 | 邮件发送可预览、可确认、可追踪 |
